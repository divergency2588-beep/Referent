# PLAN.md — План добавления функций AI-кнопок

На основании [PROJECT.MD](./PROJECT.MD).

## Цель

Подключить AI к трём кнопкам:

- **«О чём статья?»** — краткое описание содержания статьи на русском
- **«Тезисы»** — основные тезисы и ключевые идеи на русском
- **«Пост для Telegram»** — готовый пост для публикации в Telegram на русском

## Порядок действий

### Шаг 1. Подготовить промпты для каждого действия

Создать файл `lib/prompts.ts` с системными и пользовательскими промптами:

| Действие | `action` | Что должен вернуть AI |
|----------|----------|------------------------|
| О чём статья? | `summary` | 2–4 предложения: о чём статья, главная мысль |
| Тезисы | `theses` | Маркированный список из 5–10 ключевых тезисов |
| Пост для Telegram | `telegram` | Короткий пост: заголовок, анонс, 1–2 абзаца, ссылка на источник |

Промпты писать на русском, но указывать, что исходный текст статьи — на английском.

---

### Шаг 2. Добавить AI-функции в `lib/openrouter.ts`

По аналогии с `translateArticleToRussian()` создать три функции:

```text
summarizeArticle(title, content, url)  → для «О чём статья?»
extractTheses(title, content)           → для «Тезисы»
generateTelegramPost(title, content, url) → для «Пост для Telegram»
```

Общая логика:

1. Проверить, что `content` не пустой
2. Обрезать текст до `MAX_ARTICLE_CHARS` (8000 символов)
3. Вызвать `chatCompletion()` с нужным промптом
4. Вернуть текст ответа

---

### Шаг 3. Обновить API-маршрут `/api/analyze`

Файл: `app/api/analyze/route.ts`

Заменить заглушки (`PLACEHOLDER`) на реальные вызовы AI:

1. Получить `url` и `action` из запроса
2. Валидировать URL
3. Распарсить статью через `parseArticleFromUrl(url)`
4. По значению `action` вызвать нужную функцию из `lib/openrouter.ts`
5. Вернуть `{ action, result }`

---

### Шаг 4. Проверить интерфейс `ArticleForm`

Файл: `app/components/ArticleForm.tsx`

Изменения минимальны — интерфейс уже отправляет запросы на `/api/analyze` с нужным `action`. Убедиться, что:

- кнопки вызывают `handleAction("summary" | "theses" | "telegram")`
- в блоке «Результат» текст отображается как обычный текст (не JSON)
- при загрузке показывается «Генерация ответа…»

### Шаг 6. Протестировать каждую кнопку

Использовать рабочую ссылку на англоязычную статью, например:

```text
https://simple.wikipedia.org/wiki/Internet
```

| Кнопка | Ожидаемый результат |
|--------|---------------------|
| О чём статья? | Краткое описание на русском (2–4 предложения) |
| Тезисы | Список тезисов с маркерами `•` |
| Пост для Telegram | Готовый пост с эмодзи и ссылкой |

Проверить:

- корректный URL → ответ AI
- пустой / неверный URL → сообщение об ошибке
- сайт с блокировкой (Reuters) → понятная ошибка парсинга

## Риски и ограничения

- **Бесплатная модель** (`openrouter/free`) — лимит ~50 запросов/день, ответ может быть медленным
- **Длинные статьи** — обрезаются до 8000 символов; для полного анализа нужно разбивать на части
- **Блокировка сайтов** — Reuters, часть BBC не парсятся; использовать Wikipedia или Simple Wikipedia
- **Качество ответа** — зависит от модели; для production можно перейти на платную модель
